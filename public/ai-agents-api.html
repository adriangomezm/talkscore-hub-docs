API Specification

TalkScore AI Agents API

RESTful API for creating, managing, and configuring AI Agents that analyze candidate interactions across multiple interview and outreach scenarios.

Version: 1.0.0
Base URL: /api/v1
Date: 2026-03-26
Auth: Bearer Token

Endpoints

  • GET /agents — List all AI agents
  • POST /agents — Create a new AI agent
  • GET /agents/{agent_id} — Get agent details
  • PUT /agents/{agent_id} — Update an agent
  • DEL /agents/{agent_id} — Delete an agent
  • GET /agents/{agent_id}/changelog — Get agent change history
  • GET /results/{application_id} — Get assessment result for an application
  • GET /question-sets — List available question sets
  • GET /question-sets/{question_set_id} — Get question set details

Data Model: AI Agent

Core entity representing a configured AI agent for candidate analysis

Field Type Description
id uuid Unique identifier. Auto-generated on creation.
name string Display name of the agent.
description string Brief description of the agent's purpose.
category enum The use-case category for this agent.
screen_interview phone_call on_site_interview event_invitation passive_candidate_outreach status_update reminders
prompt string The instruction prompt for the AI agent. Supports candidate variable interpolation using double-brace syntax: {{candidate.first_name}}, {{candidate.email}}, etc.
question_set_id uuid | null Reference to a question set to use during the interaction. Retrieve available sets from GET /question-sets.
provider enum The external AI provider that powers this agent's underlying interaction.
elevenlabs vapi
provider_agent_id string The agent/assistant ID on the external provider's platform. For example, an ElevenLabs conversational agent ID or a Vapi assistant ID. This links the TalkScore agent to the underlying AI agent that handles the actual candidate interaction.
data_extractions DataExtraction[] Array defining what candidate attributes the agent should analyze and produce as output. At least one data extraction is required.
is_active boolean Whether the agent is active and available for use. Default: true.
created_at datetime ISO 8601 timestamp. Auto-generated.
updated_at datetime ISO 8601 timestamp. Auto-updated on changes.
Provider mapping: Each TalkScore agent wraps an external AI provider agent. The provider + provider_agent_id pair identifies which external agent powers the interaction. This allows TalkScore to orchestrate agents from different providers (e.g., ElevenLabs, Vapi) while maintaining a unified management interface.

Data Extraction Object

Each data extraction defines one attribute the agent should evaluate and produce as a result.

Field Type Description
id uuid Unique identifier. Auto-generated on creation.
candidate_attribute string Name of the candidate attribute to populate (e.g., "english_level", "years_experience", "availability_score"). This becomes the key in the analysis result.
data_type enum The data type of the produced value.
string integer
description string Human-readable description of what this data point measures and how the agent should evaluate it. This is passed to the AI as context.
number_of_values integer The number of distinct possible values this data extraction can produce. For example, a 1–5 rating scale would be 5. A yes/no classification would be 2. Min: 1.
allowed_values (string | integer)[] Explicit list of allowed output values. Length must match number_of_values when provided. If omitted, the agent can produce any value of the given data_type.
Example: A data extraction for "English proficiency level" might be: candidate_attribute: "english_level", data_type: "string", number_of_values: 6, allowed_values: ["A1", "A2", "B1", "B2", "C1", "C2"].

GET /agents/{agent_id} Get agent details

Path Parameters

ParameterTypeDescription
agent_id required uuid The agent's unique identifier

Response

200 OK — Returns the full Agent object including prompt, provider, provider_agent_id, and data_extractions.

Provider info included: The detail response always includes provider and provider_agent_id, allowing consumers to identify which external AI agent powers this TalkScore agent and make direct calls to the provider's API if needed.

Errors

StatusCondition
404Agent not found
PUT /agents/{agent_id} Update an agent

Full replacement of the agent. All required fields must be provided, including provider and provider_agent_id. To update individual data extractions, send the complete data_extractions array with the desired state.

Request Body

Same schema as POST /agents. All required fields must be included.

Data Extractions Behavior

Replace semantics: The data_extractions array is replaced entirely on update. To add a new data extraction, include it alongside the existing ones. To remove one, omit it from the array. Existing instruction IDs are preserved when the id field is included; new instructions (without id) are created.

Response

200 OK — Returns the updated full Agent object.

Errors

StatusCondition
400Validation error (same rules as POST)
404Agent not found
DEL /agents/{agent_id} Delete an agent

Permanently deletes the agent and all its data extractions. Does not delete the external provider agent — that must be managed directly on the provider's platform.

Response

204 No Content

Errors

StatusCondition
404Agent not found

Changelog Entry Object

FieldTypeDescription
id string Unique identifier for the changelog entry.
action enum The type of action performed.
created updated deleted
changed_at datetime ISO 8601 timestamp of when the change occurred.
changed_by string Identifier of the user who made the change.
changes Change[] Array of field-level changes. Empty for created actions. Each entry includes the field name, previous value, and new value.

Change Object

FieldTypeDescription

new_value any The value after the change. Type matches the field's data type.

Result Object Fields

FieldTypeDescription
agent object The agent that performed the assessment. Contains talkpush_id, provider, and identifier (the provider-side agent ID).
status string Assessment status (e.g., completed, in_progress, failed).
identifier string The conversation/session ID from the provider (e.g., ElevenLabs conversation ID).
summary string AI-generated summary of the interaction covering key points, strengths, and gaps.
application object Candidate and application information: id, first_name, last_name, email, msisdn, candidate_id, campaign_id, campaign_title, scheduled_at, others.
recording_url string URL to the audio recording of the interaction.
transcript TranscriptEntry[] Full conversation transcript. Each entry has role (bot or user), message, start (seconds), and end (seconds).
extracted_data object Key-value pairs produced by the agent's data extractions. Keys correspond to candidate_attribute names defined in the agent's configuration. Values are strings, integers, booleans, or null.
interview_insights object AI-generated analysis. Contains insights (category + notes), job_matching_score (score + explanation), and follow_up_questions (description + questions array).
created_at datetime ISO 8601 timestamp when the result was recorded.

Errors

StatusCondition
404Result not found

Validation Rules

RuleDetails
nameRequired. Must be unique per account.
descriptionRequired.
categoryRequired. Must be one of the 7 enum values.
promptRequired.
question_set_idOptional. When provided, must reference an existing question set.
providerRequired. Must be one of: elevenlabs, vapi.
provider_agent_idRequired. The external agent/assistant ID on the provider platform.
data_extractionsRequired. At least 1 data extraction per agent.
data_extractions[].candidate_attributeRequired. Must be unique within the agent.
data_extractions[].data_typeRequired. Must be "string" or "integer".
data_extractions[].descriptionRequired.
data_extractions[].number_of_valuesRequired. Integer ≥ 1.
data_extractions[].allowed_valuesOptional. When provided, length must equal number_of_values. Values must match data_type.
