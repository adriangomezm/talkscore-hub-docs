1. Webhook Processing Flow

When a TalkScore assessment call completes, Talkpush fires a webhook to TalkScore Hub. The handler must respond within 5 seconds and enqueue async processing.

Webhook Endpoint

POST /api/webhooks/call-complete
Content-Type: application/json
X-Webhook-Secret: {{WEBHOOK_SECRET}}

Expected Payload

{
  "company": "concentrix_ph",          // maps to companies.tenant_id
  "candidate_id": "12345",               // Talkpush candidate ID
  "application_id": "67890",             // Talkpush application ID
  "transcript": "Agent: Hello... Candidate: Hi...",
  "recording_url": "https://recordings.talkpush.com/..."
}

Handler Logic

1

Authenticate

Verify X-Webhook-Secret header matches WEBHOOK_SECRET env var. Reject with 401 if invalid.

2

Resolve company

Look up companies table by tenant_id = payload.company. If no match, respond 422 with {"error": "Unknown company"}.

3

Store raw call

Insert into calls table with status = 'raw' and the full payload in raw_webhook_payload.

4

Enqueue enrichment job

Add job to BullMQ enrichment queue with { callId, companyId }.

5

Respond 200

Return {"received": true, "call_id": "uuid"}. Total handler time target: <500ms.

Implementation

// src/routes/webhooks.ts
import { Router } from 'express';
import { supabaseAdmin } from '../config/supabase';
import { enrichmentQueue } from '../jobs/queue';

const router = Router();

router.post('/call-complete', async (req, res) => {
  // 1. Authenticate
  const secret = req.headers['x-webhook-secret'];
  if (secret !== process.env.WEBHOOK_SECRET) {
    return res.status(401).json({ error: 'Invalid webhook secret' });
  }

  const { company, candidate_id, application_id, transcript, recording_url } = req.body;

  // 2. Resolve company
  const { data: companyRow } = await supabaseAdmin
    .from('companies')
    .select('id')
    .eq('tenant_id', company)
    .single();

  if (!companyRow) {
    return res.status(422).json({ error: `Unknown company: ${company}` });
  }

  // 3. Store raw call
  const { data: call, error } = await supabaseAdmin
    .from('calls')
    .insert({
      company_id: companyRow.id,
      talkpush_candidate_id: candidate_id,
      talkpush_application_id: application_id,
      transcript,
      recording_url,
      status: 'raw',
      raw_webhook_payload: req.body,
    })
    .select('id')
    .single();

  if (error) {
    console.error('Failed to store call:', error);
    return res.status(500).json({ error: 'Failed to store call' });
  }

  // 4. Enqueue enrichment
  await enrichmentQueue.add('enrich', {
    callId: call.id,
    companyId: companyRow.id,
  }, {
    attempts: 3,
    backoff: { type: 'exponential', delay: 5000 },
  });

  // 5. Respond
  return res.status(200).json({ received: true, call_id: call.id });
});

2. Enrichment Worker

The enrichment worker fetches additional candidate, campaign, and agent details from the Talkpush API that aren't included in the webhook payload.

What We Fetch

DataAPI CallStored In
Candidate name, phone GET /candidates/{candidate_id} calls.candidate_name, calls.candidate_phone
Campaign name & ID GET /applications/{application_id} calls.campaign_name, calls.talkpush_campaign_id
Assessment link Resolved from assessments table via campaign_id calls.assessment_id
Agent/recruiter name GET /applications/{application_id} calls.agent_name
Call duration, date GET /applications/{application_id} calls.call_duration_seconds, calls.call_date

Worker Implementation

// src/jobs/enrichment.worker.ts
import { Worker } from 'bullmq';
import { redis } from '../config/redis';
import { supabaseAdmin } from '../config/supabase';
import { TalkpushAPI } from '../services/talkpush-api';
import { analysisQueue } from './queue';

const enrichmentWorker = new Worker(
  'enrichment',
  async (job) => {
    const { callId, companyId } = job.data;

    // Fetch the raw call
    const { data: call } = await supabaseAdmin
      .from('calls')
      .select('*')
      .eq('id', callId)
      .single();

    // Fetch company API key for Talkpush API auth
    const { data: company } = await supabaseAdmin
      .from('companies')
      .select('api_key, tenant_id')
      .eq('id', companyId)
      .single();

    const api = new TalkpushAPI(company.api_key, company.tenant_id);

    // Parallel API calls
    const [candidate, application] = await Promise.all([
      api.getCandidate(call.talkpush_candidate_id),
      api.getApplication(call.talkpush_application_id),
    ]);

    // Update call with enriched data
    const enrichmentData = { candidate, application };

    await supabaseAdmin
      .from('calls')
      .update({
        candidate_name: candidate?.name || null,
        candidate_phone: candidate?.phone || null,
        campaign_name: application?.campaign_name || null,
        talkpush_campaign_id: application?.campaign_id || null,
        agent_name: application?.agent_name || null,
        call_duration_seconds: application?.call_duration || null,
        call_date: application?.call_date || null,
        status: 'enriched',
        enrichment_data: enrichmentData,
        updated_at: new Date().toISOString(),
      })
      .eq('id', callId);

    // Resolve assessment from campaign_id (if configured)
    let assessmentId = null;
    if (application?.campaign_id) {
      const { data: assessment } = await supabaseAdmin
        .from('assessments')
        .select('id')
        .eq('company_id', companyId)
        .eq('talkpush_campaign_id', String(application.campaign_id))
        .eq('is_active', true)
        .single();

      if (assessment) {
        assessmentId = assessment.id;
        await supabaseAdmin.from('calls')
          .update({ assessment_id: assessmentId })
          .eq('id', callId);
      }
    }

    // Chain: enqueue AI analysis (include assessmentId for skill evaluation)
    await analysisQueue.add('analyze', {
      callId,
      companyId,
      assessmentId,  // null if no assessment configured
    }, {
      attempts: 3,
      backoff: { type: 'exponential', delay: 10000 },
    });
  },
  {
    connection: redis,
    concurrency: 5,  // 5 parallel enrichments
  }
);
Rate Limiting: The Talkpush API may have rate limits. The concurrency of 5 keeps us at a safe ~5 requests/second. Adjust based on actual API limits. The BullMQ rate limiter can also cap jobs per time window if needed.

3. AI Analysis Layer

The AI layer runs two parallel analyses on each call transcript via OpenRouter, extracting four dimensions of insight:

  1. TalkScore + Sentiment + Call Stage/Dropoff + Candidate Questions — Language proficiency scoring, sentiment analysis, interview stage mapping with dropoff detection, and extraction of all questions asked by the candidate
  2. Agent Quality Flags — Detects 33 flag types across agent failures, bias/fairness violations, and candidate behavior signals

Both run as parallel Promise.all calls to the same OpenRouter endpoint. Wall-clock time = max(call_1, call_2) instead of sum. The TalkScore prompt extracts all three dimensions (communication quality, sentiment, and stage/dropoff) in a single call.

Combined Analysis Prompt

Why one prompt for three dimensions? TalkScore, sentiment, and stage/dropoff analysis all depend on the same transcript context. Combining them into a single prompt avoids redundant token usage and keeps latency low (3-5s for all three outputs).

assessment call quality for recruitment. You analyze transcripts of calls

Your job is to produce FOUR analyses from a single transcript:

   agent's interview questions). For each question, capture:
   - The exact question text

- Sentiment Score: -1.00 to 1.00 (two decimals)
  - Positive: 0.25 to 1.00
  - Neutral: -0.24 to 0.24
  - Negative: -1.00 to -0.25

3a-ii. Dynamic Skills Prompt Builder

When an assessment has skills configured, the buildSkillsPromptSection() function appends a skills evaluation section to the TalkScore system prompt. This is dynamic — different assessments can evaluate different skills.

// src/services/prompts/talkscore.ts

export function buildSkillsPromptSection(skills: AssessmentSkill[]): string {
  if (!skills?.length) return '';

  const skillsList = skills.map((s, i) =>
    `  ${i + 1}. "${s.name}" (skill_id: "${s.skill_id}", max_score: ${s.max_score})
     Description: ${s.description}
     Scoring criteria: ${s.scoring_criteria}`
  ).join('\n');

  return `

5. SKILL EVALUATION — This assessment is configured to evaluate the following
   skills. For EACH skill, you must provide:
   - A score (0 to max_score for that skill)
   - A description of the candidate's performance on this skill
   - Reasoning explaining WHY you gave that score, with evidence from the transcript

SKILLS TO EVALUATE:
${skillsList}

Include a "skill_evaluations" array in your JSON response with one entry per skill,
and a "skill_summary" with a 2-3 sentence overall assessment of the candidate's
skill performance.`;
}

No skills configured? If the assessment has no skills (or the call has no linked assessment), buildSkillsPromptSection() returns an empty string. The prompt runs without the skills section, and skill_evaluations, skill_summary, and skill_average_score will be null in the database.

3b. Agent Quality Prompt

The Agent Quality Analysis uses the v4 prompt from AI INTERVIEW AGENT QUALITY ANALYSIS.md. It detects 33 flag types across three categories:

CategoryFlagsDescription
agent (12 flags) intent_misunderstood, repetition, agent_loop_suspected, poor_tone, unrelated_topic, lack_of_empathy, escalation_needed, user_confusion, vague_answers, hallucination, question_ignored, abrupt_end AI interviewer failures — loops, misunderstandings, tone issues
bias_and_fairness (9 flags) gender_bias, racial_bias, age_bias, socioeconomic_bias, disability_bias, cultural_nationality_bias, language_proficiency_bias, accent_and_dialect_bias, interview_format_bias Discrimination and fairness violations
candidate (4 flags) requested_escalation, used_profanity, inappropriate_content, privacy_concern_raised Candidate behavior signals
Critical flags that trigger has_critical_flags = TRUE: agent_loop_suspected, hallucination, and any bias_and_fairness flag. These surface prominently in the dashboard for immediate review.

Analysis Worker (Dual Parallel)

// src/jobs/analysis.worker.ts
import { Worker } from 'bullmq';
import { redis } from '../config/redis';
import { supabaseAdmin } from '../config/supabase';
import { OPENROUTER_CONFIG, callOpenRouter } from '../config/openrouter';
import { TALKSCORE_SYSTEM, TALKSCORE_USER, buildSkillsPromptSection } from '../services/prompts/talkscore';
import { QUALITY_PROMPT } from '../services/prompts/agent-quality';

// Critical flag IDs that trigger has_critical_flags
const CRITICAL_FLAGS = new Set([
  'agent_loop_suspected', 'hallucination',
  'gender_bias', 'racial_bias', 'age_bias',
  'socioeconomic_bias', 'disability_bias',
  'cultural_nationality_bias', 'language_proficiency_bias',
  'accent_and_dialect_bias', 'interview_format_bias',
]);

const analysisWorker = new Worker(
  'analysis',
  async (job) => {
    const { callId, companyId, assessmentId } = job.data;

    // Fetch transcript, company settings, and assessment skills (if configured)
    const queries = [
      supabaseAdmin.from('calls').select('transcript').eq('id', callId).single(),
      supabaseAdmin.from('companies').select('settings').eq('id', companyId).single(),
    ];
    if (assessmentId) {
      queries.push(
        supabaseAdmin.from('assessments').select('skills, name').eq('id', assessmentId).single()
      );
    }
    const results = await Promise.all(queries);
    const call = results[0].data;
    const company = results[1].data;
    const assessment = results[2]?.data || null;

    if (!call?.transcript) throw new Error(`No transcript for call ${callId}`);

    const model = company?.settings?.openrouter_model || OPENROUTER_CONFIG.defaultModel;

    // ═══ BUILD DYNAMIC PROMPT WITH SKILLS (if assessment configured) ═══
    const skillsSection = assessment?.skills?.length
      ? buildSkillsPromptSection(assessment.skills)
      : '';
    const systemPrompt = TALKSCORE_SYSTEM + skillsSection;

    // ═══ RUN BOTH ANALYSES IN PARALLEL ═══
    const talkscoreStart = Date.now();
    const qualityStart = Date.now();

    const [talkscoreResult, qualityResult] = await Promise.all([
      // Analysis 1: TalkScore + Sentiment + Stages + Skills
      callOpenRouter({
        model,
        systemPrompt,
        userPrompt: TALKSCORE_USER.replace('{transcript}', call.transcript),
        maxTokens: 3000,  // increased for skill evaluations
      }),
      // Analysis 2: Agent Quality Flags
      callOpenRouter({
        model,
        systemPrompt: QUALITY_PROMPT,     // Full v4 prompt from .md file
        userPrompt: call.transcript,
        maxTokens: 2000,
      }),
    ]);

    // Parse results
    const ts = talkscoreResult.parsed;        // TalkScore JSON
    const qf = qualityResult.parsed;           // Quality flags JSON

    // Extract flags (handle array wrapper from v4 prompt)
    const qualityData = Array.isArray(qf) ? qf[0] : qf;
    const flags = qualityData?.flags || [];
    const hasCritical = flags.some(f => CRITICAL_FLAGS.has(f.flag_id));

    // Store combined analysis
    await supabaseAdmin.from('analyses').insert({
      call_id: callId,
      company_id: companyId,

      // TalkScore
      talkscore_overall: ts.talkscore_overall,
      talkscore_breakdown: ts.talkscore_breakdown,
      communication_issues: ts.communication_issues,
      candidate_issues: ts.candidate_issues,
      agent_issues: ts.agent_issues,

      // Sentiment
      sentiment_overall: ts.sentiment_overall,
      sentiment_score: ts.sentiment_score,
      sentiment_breakdown: ts.sentiment_breakdown,

      // Call Stage / Dropoff
      last_stage_reached: ts.last_stage_reached,
      total_questions_asked: ts.total_questions_asked,
      total_questions_expected: ts.total_questions_expected,
      call_completed: ts.call_completed,
      dropout_reason: ts.dropout_reason,
      stages: ts.stages,

      // Candidate Questions
      candidate_questions: ts.candidate_questions || [],
      candidate_question_count: ts.candidate_question_count || 0,

      // Skill Evaluations (only present if assessment has skills configured)
      skill_evaluations: ts.skill_evaluations || null,
      skill_summary: ts.skill_summary || null,
      skill_average_score: ts.skill_evaluations?.length
        ? +(ts.skill_evaluations.reduce((sum, s) =>
            sum + (s.score / s.max_score) * 10, 0) / ts.skill_evaluations.length).toFixed(2)
        : null,

      // Agent Quality Flags
      quality_flags: flags,
      quality_flag_count: flags.length,
      quality_summary: qualityData?.summary || null,
      has_critical_flags: hasCritical,

      // AI metadata (tracked per-analysis)
      talkscore_model: model,
      talkscore_tokens: talkscoreResult.totalTokens,
      talkscore_duration_ms: talkscoreResult.durationMs,
      quality_model: model,
      quality_tokens: qualityResult.totalTokens,
      quality_duration_ms: qualityResult.durationMs,
      raw_talkscore_response: ts,
      raw_quality_response: qualityData,
    });

    // Update call status
    await supabaseAdmin.from('calls').update({
      status: 'analyzed',
      updated_at: new Date().toISOString(),
    }).eq('id', callId);
  },
  {
    connection: redis,
    concurrency: 3,   // 3 parallel analysis jobs (each makes 2 API calls)
    limiter: {
      max: 30,         // max 30 jobs/min = 60 OpenRouter calls/min
      duration: 60000,
    },
  }
);

OpenRouter Helper

// src/config/openrouter.ts — shared caller
export async function callOpenRouter({ model, systemPrompt, userPrompt, maxTokens }) {
  const start = Date.now();
  const response = await fetch(`${OPENROUTER_CONFIG.baseURL}/chat/completions`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.OPENROUTER_API_KEY}`,
      'Content-Type': 'application/json',
      'HTTP-Referer': 'https://talkscore-hub.talkpush.com',
      'X-Title': 'TalkScore Hub',
    },
    body: JSON.stringify({
      model,
      messages: [
        { role: 'system', content: systemPrompt },
        { role: 'user', content: userPrompt },
      ],
      max_tokens: maxTokens,
      temperature: 0.1,
      response_format: { type: 'json_object' },
    }),
  });

  if (!response.ok) {
    throw new Error(`OpenRouter ${response.status}: ${await response.text()}`);
  }

  const result = await response.json();
  return {
    parsed: JSON.parse(result.choices[0].message.content),
    totalTokens: (result.usage?.prompt_tokens || 0) + (result.usage?.completion_tokens || 0),
    durationMs: Date.now() - start,
  };
}

Model Fallback Strategy

If the primary model fails, the BullMQ worker retries with exponential backoff. On the 3rd attempt, switches to the fallback model:

  1. Attempt 1: anthropic/claude-sonnet-4-20250514 (primary)
  2. Attempt 2: Same model, after 10s backoff
  3. Attempt 3: openai/gpt-4o (fallback)

Per-company model override via companies.settings.openrouter_model.

5. CRM Bridge Login (SSO via PAM)

Users access TalkScore Hub from the Talkpush CRM. The CRM generates a single-use token (60s TTL), redirects to TalkScore Hub, which validates it with PAM and creates a local session.

Login Flow

  1. User clicks "TalkScore Hub" in Talkpush CRM
  2. CRM calls PAM POST /auth/token → gets single-use token
  3. CRM redirects to https://talkscore-hub.replit.app/auth/crm-bridge?token=xxx
  4. TalkScore Hub validates token with PAM POST /auth/validate
  5. TalkScore Hub creates/updates local profile + session
  6. User is redirected to dashboard

Implementation

// src/auth/crm-bridge.ts
import { Router } from 'express';
import { supabaseAdmin } from '../config/supabase';

const PAM_URL = 'https://product-access-manager.vercel.app/api/v1';
const router = Router();

router.get('/crm-bridge', async (req, res) => {
  const { token } = req.query;

  if (!token) {
    return res.status(400).send('Missing token');
  }

  try {
    // Validate token with PAM
    const pamResponse = await fetch(`${PAM_URL}/auth/validate`, {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${process.env.ACCESS_MANAGER_API_KEY}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ token }),
    });

    if (pamResponse.status === 410) {
      return res.status(410).send('Login link expired. Please try again from the CRM.');
    }
    if (!pamResponse.ok) {
      return res.status(pamResponse.status).send('Authentication failed');
    }

    const user = await pamResponse.json();

    // Resolve or create local company
    let { data: company } = await supabaseAdmin
      .from('companies')
      .select('id')
      .eq('tenant_id', user.client.tenant_id)
      .single();

    if (!company) {
      // Auto-create company from PAM data
      const { data: newCompany } = await supabaseAdmin
        .from('companies')
        .insert({
          name: user.client.name,
          tenant_id: user.client.tenant_id,
          status: 'active',
        })
        .select('id')
        .single();
      company = newCompany;
    }

    // Create or update Supabase Auth user
    // (use admin API since CRM Bridge users don't sign up normally)
    let authUser;
    const { data: existingUsers } = await supabaseAdmin.auth.admin
      .listUsers();
    const existing = existingUsers.users
      .find(u => u.email === user.email);

    if (existing) {
      authUser = existing;
    } else {
      const { data } = await supabaseAdmin.auth.admin.createUser({
        email: user.email,
        email_confirm: true,
        user_metadata: { full_name: user.full_name },
      });
      authUser = data.user;
    }

    // Upsert profile (PAM standard)
    await supabaseAdmin
      .from('profiles')
      .upsert({
        id: authUser.id,
        email: user.email,
        full_name: user.full_name,
        auth_source: 'crm_bridge',  // NOT synced back to PAM
      }, { onConflict: 'id' });

    // Upsert user role for this company
    await supabaseAdmin
      .from('user_roles')
      .upsert({
        user_id: authUser.id,
        client_id: company.id,
        role: user.role,  // 'admin' | 'manager' | 'user'
      }, { onConflict: 'user_id,client_id' });

    // Generate a Supabase session for the user
    const { data: session } = await supabaseAdmin.auth.admin
      .generateLink({ type: 'magiclink', email: user.email });

    // Set session cookie and redirect to dashboard
    res.cookie('sb-access-token', session.properties.hashed_token, {
      httpOnly: true,
      secure: true,
      sameSite: 'lax',
      maxAge: 86400 * 1000,  // 24 hours
    });

    return res.redirect('/');
  } catch (error) {
    console.error('CRM Bridge Login failed:', error);
    return res.status(500).send('Authentication failed');
  }
});
Key behavior: CRM Bridge users are created with auth_source = 'crm_bridge'. The PAM sync service (Section 7) explicitly skips these users to prevent infinite sync loops.

6. Real-Time Dashboard Updates

The dashboard receives live updates as calls are processed using Supabase Realtime. When a call's status changes (raw → enriched → analyzed), the dashboard updates automatically.

Supabase Realtime Configuration

Enable Realtime on the calls and analyses tables in Supabase Dashboard → Database → Replication.

Frontend Hook

// client/src/hooks/useRealtimeUpdates.ts
import { useEffect } from 'react';
import { supabase } from '../config/supabase';

export function useRealtimeCalls(companyId: string, onUpdate: (call: any) => void) {
  useEffect(() => {
    const channel = supabase
      .channel('calls-realtime')
      .on(
        'postgres_changes',
        {
          event: '*',          // INSERT, UPDATE
          schema: 'public',
          table: 'calls',
          filter: `company_id=eq.${companyId}`,
        },
        (payload) => {
          onUpdate(payload.new);
        }
      )
      .subscribe();

    return () => {
      supabase.removeChannel(channel);
    };
  }, [companyId, onUpdate]);
}

export function useRealtimeAnalyses(companyId: string, onAnalysis: (analysis: any) => void) {
  useEffect(() => {
    const channel = supabase
      .channel('analyses-realtime')
      .on(
        'postgres_changes',
        {
          event: 'INSERT',
          schema: 'public',
          table: 'analyses',
          filter: `company_id=eq.${companyId}`,
        },
        (payload) => {
          onAnalysis(payload.new);
        }
      )
      .subscribe();

    return () => {
      supabase.removeChannel(channel);
    };
  }, [companyId, onAnalysis]);
}

Client Discovery Endpoint

// src/routes/access-manager.ts
import { Router } from 'express';
import { supabaseAdmin } from '../config/supabase';

const router = Router();

// PAM pulls client list
router.get('/clients', async (req, res) => {
  const apiKey = req.headers.authorization?.replace('Bearer ', '');
  if (apiKey !== process.env.ACCESS_MANAGER_API_KEY) {
    return res.status(401).json({ error: 'Unauthorized' });
  }

  const { data: companies } = await supabaseAdmin
    .from('companies')
    .select('name, tenant_id, status')
    .not('tenant_id', 'is', null)
    .order('name');

  return res.json({
    clients: (companies || []).map(c => ({
      name: c.name,
      tenant_id: c.tenant_id,
      status: c.status,
    })),
  });
});

// PAM pulls user list
router.get('/users', async (req, res) => {
  const apiKey = req.headers.authorization?.replace('Bearer ', '');
  if (apiKey !== process.env.ACCESS_MANAGER_API_KEY) {
    return res.status(401).json({ error: 'Unauthorized' });
  }

  // Fetch users with their roles and company tenant_ids
  // EXCLUDE crm_bridge users (they came FROM PAM, don't sync back)
  const { data: profiles } = await supabaseAdmin
    .from('profiles')
    .select(`
      email, full_name,
      user_roles (
        role,
        companies ( tenant_id )
      )
    `)
    .neq('auth_source', 'crm_bridge');

  const users = (profiles || []).flatMap(p =>
    (p.user_roles || [])
      .filter(r => r.companies?.tenant_id)
      .map(r => ({
        email: p.email,
        full_name: p.full_name,
        role: r.role,
        tenant_id: r.companies.tenant_id,
      }))
  );

  return res.json({ users });
});

Push Sync Helper

Called after user create/update/delete in TalkScore Hub admin panel:

// src/auth/pam-sync.ts

const PAM_URL = 'https://product-access-manager.vercel.app/api/v1';

export async function syncUserToPAM(
  action: 'register' | 'delete',
  userData: { email: string; full_name?: string; role?: string; tenant_id: string }
) {
  try {
    const response = await fetch(`${PAM_URL}/users/register`, {
      method: action === 'delete' ? 'DELETE' : 'POST',
      headers: {
        'Authorization': `Bearer ${process.env.ACCESS_MANAGER_API_KEY}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify(userData),
    });

    if (!response.ok) {
      console.error(`PAM sync failed (${response.status}):`, await response.text());
    }
  } catch (err) {
    // Fire-and-forget: PAM is secondary, don't block product operations
    console.error('PAM sync error:', err.message);
  }
}
Fire-and-forget: PAM sync NEVER blocks TalkScore Hub operations. If PAM is down, log the error and move on. The user's local account still works. PAM can catch up via the pull-based /api/access-manager/users endpoint.

// Re-run analysis for a specific call
router.post('/reprocess/:callId', requireAdmin, async (req, res) => {

  // Reset status
  await supabaseAdmin
    .from('calls')
    .update({ status: 'enriched', error_message: null })
    .eq('id', callId);

  // Delete old analysis if exists
  await supabaseAdmin
    .from('analyses')
    .delete()
    .eq('call_id', callId);

  // Re-enqueue
  const { data: call } = await supabaseAdmin
    .from('calls')
    .select('company_id')
    .eq('id', callId)
    .single();

  await analysisQueue.add('analyze', {
    callId,
    companyId: call.company_id,
  });

  res.json({ reprocessing: true, call_id: callId });
});
